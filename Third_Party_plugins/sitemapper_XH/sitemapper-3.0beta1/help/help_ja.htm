Sitemapper_XH - a CMSimple_XH Plugin

Sitemapper_XHは自動的にXML-サイトマップを作成します。サイトマップの詳細についてはsitemaps.orgを参照してください

必要環境

Sitemapper_XHはCMSimple_XHためのプラグインです。CMSimple_XH≥1.6.3とPHP≥5.4.0が必要です.

back to top

インストール

The installation is done as with many other CMSimple_XH plugins. See the CMSimple_XH wiki for further details.

  1. Backup the data on your server.
  2. Unzip the distribution on your computer.
  3. Upload the whole directory sitemapper/ to your server into the CMSimple_XH plugins directory.
  4. Set write permissions for the subdirectories config/ und languages/.
  5. Switch to "Sitemapper" in the back-end to check if all requirements are fulfilled.
back to top

Settings

The plugin's configuration is done as with many other CMSimple_XH plugins in the website's back-end. Select Plugins → Sitemapper.

You can change the default settings of Sitemapper_XH in "Config". Hints for the options will be displayed when hovering over the help icons with your mouse.

Localization is done under "Language". You can translate the character strings to your own language (if there is no appropriate language file available), or customize them according to your needs.

使い方

Sitemapper_XHを使用すると、ウェブサイトのサイトマップを利用できます。インストールされている言語ごとに、サイトマップ索引文書と追加のサイトマップ文書があります。サイトマップは自動で作成され、ファイルとして保存されません。

「非公開ページ」はサイトマップに含まれません。「隠しページ」は、デフォルトでは含みませんが、「Hidden page(隠しページを掲載しない）」のチェックを外すと、サイトマップに掲載されるようになります。
プラグイン管理（　http://www.example.com/?sitemapper&normal ）の【利用可能なサイトマップ】で、サイトマップのリスト　index　と、（日本語環境ならば）　ja　が表示されます .
　（サイトが多言語化される場合は、言語に対応したenやduが表示される？　未確認）

●robots.txt　に追加する場合・・ index を使います

Sitemap: http://www.example.com/?sitemapper_index

●Googleウェブマスターツール等への登録・・ ja を使います

http://www.example.com/?sitemapper_sitemap

※各ページの編集画面のエディタのタブに「Sitemap｣タブが追加されます。
各ページについて、プラグイン設定で定義されている "changefreq（ページの更新頻度）"と "priority（URLの優先順位）"のデフォルト設定を上書きすることができます。
この設定の詳細については、「XML tag　definitions（XMLタグ定義）」を参照してください。

制限事項

Sitemap_XHは、サイトマップがどのように要求されたかに応じて、サイトマップにURLを生成します。 CMSimple_XHインストールがhttp://example.com/、https://www.example.com/、http://www.example.com/index.phpなどからアクセス可能な場合、URLはそれぞれ異なる場合があります。これを避けるには、適切なリダイレクトを設定する必要があります。例えばSeo_XHプラグインで簡単に行うことができます。
・・注）Seo_XH　プラグインは、未完成です（2017/08現在）⇒github

Sitemap_XHは、CMSimple_XHのコンテンツの実際のページのみを一覧表示します。プラグインによって管理される追加のコンテンツ（例：フォーラムやブログは、サイトマップには含まれません）
